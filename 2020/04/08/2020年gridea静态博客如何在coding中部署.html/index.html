前言

随着一月份的coding改革，原来网上关于Gridea博客有关coding的设置已经太老了。

如今Coding已经变成团队协作的工具，相对于之前的操作虽然变化不大，但是坑还是挺容易踩到的。

本文Gridea版本：0.9.2

Gridea

Gridea 是一个静态博客写作客户端，帮助你更容易地构建并管理博客或任何静态站点。

官网：https://gridea.dev/

Coding

一站式 DevOps，提升研发效能全新 CODING 涵盖了软件开发从构想到交付的一切所需，使研发团队在

云端高效协同，实践敏捷开发与 DevOps，提升软件交付质量与速度。

官网：https://coding.net/

开始

首先，我们需要现在Coding中创建一个新项目，项目类型选择代码托管项目。

之后我们看到项目基本信息填写处，这里比较重要的就是项目名称，这是我们接下来要在Gridea中设置的，其他信息就随意发挥了。

新建完我们进入项目页，此时我们需要以此点击以下操作来进行静态pages部署。

项目设置（项目页左下角）>功能开关>打开构建与部署

紧接着，我们返回项目页，在构建与部署中找到静态网站，进入之后点击立即发布静态网站（请注意，发布静态网站需要进行账号实名认证）

静态网站的网站名称之类填写随意发挥，部署来源和触发机制保持不变。创建完静态网站后，点击立即部署按钮开始进行网站部署。

注意，此时的访问地址将会是我们在Gridea中设置的域名。如果你有自己的域名，可以在静态网站设置中添加。

Gridea客户端设置

首先我们在Gridea客户端Coding设置中能够看到的有：

  • 域名
  • 仓库名称
  • 分支
  • 仓库用户名
  • 邮箱
  • 令牌用户名
  • 令牌
  • CNAME

而以上除了邮箱和CNAME以外，其他都是我们必须设置的选项，邮箱地址随意填写。

域名

这里是我们在Coding静态网站设置的域名，比如我绑定了我的域名094250.xyz，那我就填写094250.xyz。

域名之前的https://和http://，如果你静态网站开启了强制https，那你可以选https://

仓库名称

这里的仓库名称是指我们在Coding中创建的项目的项目名，在我们文章开始部分有讲到。

分支

这里我们默认是master，我们如果没有其他分支的话，我们默认是填写master。

仓库用户名

请注意，这是个易错内容

在我们Coding新版中找不到仓库用户名（我找不到），其实这个仓库用户名就是我们在注册Coding时填写的团队域名内容框里填写的内容。如果忘了，可以通过在仓库设置中找到https://e.coding.net/ /**/，这里的就是我们的仓库用户名。

操作步骤：项目页面>代码仓库>仓库设置>仓库名称，如下图，以下光标内容即仓库用户名：

令牌

这里我们回到Coding，在页面的右上角有我们的头像，在鼠标移动到头像会出现二级导航，选择个人设置，点击进入之后，点击访问令牌。在这里我们新建令牌。

令牌的描述随便填写。权限我们只需要选择一个project:depot。然后创建完成，出现令牌串码，复制串码填写到Gridea中。

令牌用户名

在访问令牌页面，如下图，红框框里的内容，就是令牌用户名，将它填入Gridea中的令牌用户名就行了：

最后

我们设置完在Gridea客户端中先点击保存，再点击检测远程连接。

此时，如果我们看到远程连接成功，那么我们就可以开始写博客了！设置成功的同学，别忘了在评论留下作业哦！查看原帖
